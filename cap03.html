Aplicando contadores a los títulos h2, h3, h4, etc.

Clase .Contenido define un contador con nombre "temarios" para cada h2 y un "subtemarios" para cada h3. .Contenido realiza un counter.reset al contador "temarios", así como h2:before realiza un reset al contador "subtemarios" y h3:before resetea el contador "subtema-h4". En h2::before realiza un incremento del contador "temarios", mientras en h3::before se incrementa "subtemarios" 


    .Contenido {
      counter-reset: temarios;
      padding: 0 1rem;
    }
    .Contenido h2 {
      counter-reset: subtemahTres;
      counter-increment: temarios;
    }
    .Contenido h3 {
      counter-reset: subtemahCuatro;
      counter-increment: subtemahTres;
    }
            

    .Contenido h4 {
      counter-increment: subtemahCuatro;
    }
    .Contenido h2::before {
      content: counter(temarios) ".- ";
    }
    .Contenido h3::before {
      content: counter(temarios) "." counter(subtemahTres) ".- ";
    }
    .Contenido h4::before {
      content: counter(temarios) "." counter(subtemahTres) "."
        counter(subtemahCuatro) ".- ";
    }

Se puede tener tres tipos de sombras: a la caja, al texto y el filtro drop-shadow

Sombras a las cajas - (box-shadow)

box-shadow: se determina con distancia en x, distancia en y, el desenfoque radial conocido como blur-radius (no negativo), la expansión del desenfoque spread-blur (+|- + expande más, - reduce tamaño, 0 sombra mismo tamaño que el elemento), el color (opcional o lo calcula dependiendo del contenido) y adicionalmente si la sombra es interna o externa inset/outset, solo admite inset, outset es por default (no se escribe). Si no se especifica el color toma el color de texto del contenido de la caja. Color de texto por default es negro #000

Dos, tres, o cuatro valores <length>. Si solo dos valores son definidos, son interpretados como valores <offset-x> <offset-y>. Si un tercer valor es dado, es interpretado como un <blur-radius>. Si un cuarto valor es dado, es interpretado como un <spread-radius>. Opcionalmente, la palabra clave inset. Opcionalmente, el valor <color>. 


            box-shadow: x y blur-radius spread-blur color inset/outset

        height: 50px;

Filtro de sombras - (drop-shadow)

La propiedad filter permite invocar algunas funciones entre ellas drop-shadow(). La propiedad CSS filter dota de efectos gráficos como el desenfoque o cambio de color en la representación antes de que se muestre el elemento. Los filtros se utilizan comúnmente para ajustar la representación de imágenes, fondos o bordes. También se pueden combinar varias funciones aplicando como filtro: 


    filter: url("filters.svg#filter-id");
    filter: blur(5px);  /* Desenfoque Gaussiano, no acepta porcentajes */
    filter: brightness(0.4);  /* +|- brillante 0% negro, 100% sin cambios default 1 = 100%*/
    filter: contrast(200%);   /* +|- contraste, 0% imagen gris, 100% sin cambios, default 1=100%, >100% mayor contraste*/
    filter: drop-shadow(16px 16px 20px blue); /* Efecto sombra revisar box-shadow y text-shadow. aplica a imagenes con transparencia alfa (png) */
    filter: grayscale(50%); /* 0% (sin cambios) hasta 100% (escala de grises)*/
    filter: hue-rotate(90deg); /* 0deg (sin cambios) hasta 360deg (aplicando rotacion de tonos)*/
    filter: invert(75%);  /* 0% (sin cambios) hasta 100% (invierte colores de imagen de acuerdo a proporción) */
    filter: opacity(25%); /* 0% (completamente transparente) hasta 100% (sin cambios), igual a propiedad opacity*/
    filter: saturate(30%);  /* 0% (completamente sin saturación) hasta 100% (sin cambios), >100% permitido y genera sobre saturación*/
    filter: sepia(60%); /* 0% (sin cambios) hasta 100% (completamente sepia)*/
    /* Apply multiple filters */
    filter: contrast(175%) brightness(3%);
    
    .drop-shadow {
      box-shadow: 0.4rem 0.3rem 0.7rem -0.15rem #0006,

      filter: drop-shadow(0.2rem 0.5rem 0.2rem red) sepia(0%) contrast(200%);
    }

Filtros (filter)

Los filtros se aplican sobre imagenes, sobre texto son poco apreciables.

blur(15px) valor de desenfoque, no negativo, no en porcentaje, si en px o rem.
brightness() entre 0 (oscuro total) y 1 (brillo total), mayor a 1 sobre exposición a la luz como efecto de quemar la imagen.
contrast() entre 0 (recuadro gris) hasta 1 (sin contraste), mayores a 1 se sobreexpone a oscuridad como efecto de quemar la imagen
saturate() saturación 0(Totalmente escala casi grises) hasta 1 (sin saturacion, colores propios de la imagen), > 1 muchos rojos como imagen quemada.
grayscale() entre 0 (no aplicar) hasta 1 (escala grises)
sepia() entre 0 (no aplicar) hasta 1 (total sepia - amarillo)
hue-rotate(150deg) Tono matiz de color de acuerdo a HSLA_hue_wheel_with_turn la escala de colores HSLA.
invert() invierte colores de imagen 0% (sin cambios) 100%(total inversión de colores) o con valores de 0 a 1, como un lo tienes o no 0|1.
opacity() Opacidad 0(total opacidad) hasta 1(total visible).
drop-shadow Similar a box-shadow requiere offset-x, offset-y, opcionales son blur-radius no negativo desde 0 (nitido) hasta 1. spread-radius positivo la sombra se expande, negativos se contrae y color

Modos de Mezcla (mix-blend-mode)

Efecto de tener dos o más capas y generar el efecto de fusión. En el div.box-card.relative.blend-modes, notese que el div contendor tiene la clase blend-modes para aplicar a la img, se tienen dos img, la segunda con .box-card-absolute
Los valores posibles de mix-blend-mode son:
normal por default, no aplica mezcla alguna, multiply multiplica fondo y contenido, reemplaza el fondo, resultante siempre es tan oscuro como el fondo, screen multiplica fondo y contenido complementa resultado, genera contenido mas brillante que color de fondo, overlay superposicion, multiplica o filtra el contenido según el color de fondo. Este es el inverso del modo de mezcla de luz dura hard-light, hard-light dependiendo del color de contenido este atributo lo filtrará o multiplicará, darken el fondo es reemplazado por el contenido en las zonas más oscuras, sino, se deja el fondo, lighten el fondo se reemplaza con el contenido donde el contenido es más claro, color-dodge ilumina color de fondo para reflejar el color del contenido, color-burn oscurece el fondo para reflejar el color natural del contenido, soft-light dependiendo del color del contenido, este lo oscurecerá o aclarará. difference resta el más oscuro de los dos colores del color más claro, exclusion similar a diferencia pero menor contraste, hue crea color con tonalidad del contenido combinado con saturación y luminosidad del fondo, saturation crea color con saturación del contenido combinada con matiz y luminosidad del fondo, color crea color con matiz y saturación del contenido y luminosidad del fondo, luminosity crea color con luminosidad del contenido y matiz y saturación del fondo. Inverso de color.

El slide hereda de .slides el height
Se debe agregar al contenedor cuya clase es slides la propiedad scroll-snap-type el cual recibe dos valores, el primer valor indicara que eje se quiere controlar, x o inline para desplazamiento horizontal, y o block para desplazamiento vertical, both para ambas direcciones; el segundo parametro indica si es mandatorio (un ligero toque del scroll del mouse y cambia de diapositiva) o por proximidad hacia la diapositiva mas cercana. scroll-snap-type por default es none.
La clase .slide debe contener la propiedad scroll-snap-align: center

skew (deformar o sesgar) mediante las lineas perpendiculares o paralelas del objeto, no existe en 3D, skewX( [+|-] 0deg - 360deg ) con negativos hacia la derecha, skewY( [+|-] 0deg - 360deg) con negativos hacia abajo y su shorthand skew( en x 0deg-360deg, en y 0deg-360deg).

Propiedades de Animación en CSS

animation-timing-function Corresponde a la bezier como se ejecuta la animación pudiendo ser ease, ease-in, ease-out, ease-in-out, linear o cubic-bezier, igual que en timing-function de la propiedad transition.
animation-delay Es el retardo para inicio de animaciones por default es 0, puede expresarse en milisegundos o en segundos.
animation-iteration-count Cantidad de veces que se repite la animación, por default es 1. Puede ser infinite.
animation-direction Posibles valores alternate (alterna los inicios entre from y to uno a uno), reverse (empieza en to y termina en from), alternate-reverse (alterna los inicios entre to y from uno a uno), normal (default)
animation-fill-mode posibles valores backwards (al inicio de la animación se queda con los estilos iniciales de la animación), forwards (al fin de la animación se queda con los estilos finalesd de la animación), both (combina forwards y backwards), none (default)
