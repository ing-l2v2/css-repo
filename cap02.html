Selectores de hermanos

Selector de hermanos (en General)

Selectores que estan en el mismo nivel del elemento de referencia pero todos a continuacion del elemento que tiene la class, se invocan con una ~ entre el nombre de la clase y la etiqueta:
.hermanos-general ~ li { background-color: thistle; }
, el elemento con la clase es Verano, seleccionara todos los hermanos excepto Verano.

  • Verano

Selector de hermanos (Adyacente)

Selectores que estan en el mismo nivel del elemento de referencia pero solo uno a continuacion del elemento que tiene la class, se invocan con un + entre el nombre de la clase y la etiqueta:
.hermanos-general + li { background-color: thistle; }
, el elemento con la clase es Verano, seleccionara solo el hermano adyacente siguiente.

Selectores de Atributos

Selector para los elementos contenidos que cumplen condicion de un atributo, hasta por un valor atributo, parte del contenedor especificado por la clase
.selectores-atributos a[href] { color: orangered; }
.selectores-atributos a[target = "_blank"] { font-weight: bold; }
Se puede usar el caracter comodin * para coincidencias que contenga el texto en alguna parte del atributo
.selectores-atributos a[href *= "indice"] { font-size: 24px; background-color: navajowhite; color: black; }
Comodin ^ (circunflejo) para coincidencias que comienzen con el texto referido
.selectores-atributos a[href ^= "http:"] { font-size: 25px; background-color: navy; color: white; text-decoration: none; }
El comodin $ aplica si el texto en cuestion esta al final
.selectores-atributos a[href $= ".com"] { font-size: 18px; background-color: navy; color: white; text-decoration: none; }
El comodin ~ aplica si el texto forma parte de una lista separada por espacios
.selectores-atributos a[class ~= "cursos"] { text-decoration: none; }
El comodin | (pipe) aplica si el texto forma parte de una lista separada por espacios y que despues del texto venga un guion medio - ejemplo menu-item en las clases
.selectores-atributos a[class |= "menu"] { font-style: italic; }

El Algoritmo CSS

Es la forma en que el navegador aplica los estilos al documento HTML. Es vital entender estos conceptos para entender como se aplican y en ocasiones se sobreescriben reglas CSS: (1) La Cascada, (2) La Especificidad y (3) La Herencia.

1. La Cascada

Es el mecanismo que tiene el navegador web para ir aplicando los estilos tomando tres aspectos particulares: El origen del codigo (de donde viene) 1 User Agent (por defecto aplica el navegador) 2 Personalizaciones del usuario (SO y navegador Ej. Modo oscuro y el zoom) 3 Estilos del autor mediante las hojas de estilos. Luego se aplica la especificidad y finalmente el orden de aparición.

2. La Especificidad

El peso para css que tiene un selector cuando hay conflicto de estilos, se calcula de la siguiente forma: 

  1. Etiquetas y pseudoelementos      0.0.0.1
  2. Clases atributos y pseudoclases  0.0.1.0
  3. Identificadores                              0.1.0.0
  4. Estilos en linea                             1.0.0.0
  5. !important            (Rompe la especifidad)
            blockquote {
              background-color: burlywood;
            }
            #cita-marco {
              background-color: lightgreen;
            }
            blockquote.cita-marco {
              background-color: tomato;
            }
            #cita-marco {
              background-color: moccasin !important;
            }
            .cita-marco {
              background-color: cornflowerblue;
            }
Las reglas css del codigo previo, ayudan a entender la especificidad.

Se tolerante con los demás y estricto contigo mismo - Marco Aurelio

3. La Herencia

La herencia es la capacidad de un selector de obtener (heredar) los valores de sus ancestros mas cercanos, para aplicarla se usa el valor inherit, si queremos evitarla se puede asignar otro valor o inicializar la propiedad en cuestion con el valor initial.
Ciertas propiedades no se heredan: Bordes, Fondo, padding, margin.
Se heredan todas las propiedades relacionadas a la tipografía, border-collapse, border-spacing, caption-side, cursor, direction, empty-cells, letter-spacing, line-height, list-style-image, list-style-position. Referncia sobre herencia en Inheritance - web.dev, Español

Modelo de caja

Ver en las referencias el articulo sobre el modelo de caja.
Este tema es tratado en un nuevo html llamadao modelo-caja.html El modelo de caja
Se revisa border, padding y margin: (Sentido manecillas del reloj desde top) con un valor aplica a los cuatro lados, con dos valores aplica el primer valor para top, bottom y el segundo valor para left, right. Con tres valores aplica (1) top, (2) left, right y (3) bottom. Con 4 valores aplica top, right, buttom, left.

Cajas de linea vs cajas de bloque

Comparación entre Cajas de linea y cajas de bloque
Cajas de linea

  • Ocupan el espacio necesario para mostrar su contenido
  • No tienen dimensiones modificables (width, height)
  • Permiten otros elementos a su lado
  • Padding y margin solo empujan elementos adyacentes en horizontal, NUNCA EN VERTICAL, nunca se debe aplicar padding verticales a una caja de linea
Cajas de bloque
  • Ocupan todo el ancho disponible, lo que genera saltos de linea
  • Tienen dimensiones modificables (width, height)
  • No permiten otros elementos a su lado (Aunque especifique un ancho, siguen ocupando todo el espacio disponible a lo ancho generando saltos de linea

Propiedad display

display: none El elemento se elimina por completo, como si no estuviera en el código HTML
display: inline El elemento se convierte en de linea, se comporta como texto simple. height y width no tendrán efecto.
display: block El elemento se convierte en de bloque, comienza en nueva linea y ocupa todo el ancho.
display: inline-block El elemento se convierte en mixto comparte propiedades de elemento de linea y de bloque, comportandose como texto simple que se inserta en un bloque de texto y se puede aplicar width y height.
display: list-item El elemento se comporta como un elemento de lista, de bloque con viñeta. EN LA PRACTICA NO SE APLICA
display: table El elemento se comporta como una tabla, su contenido y elementos secundarios se comportan como celdas de la tabla, de bloque. EN LA PRACTICA NO SE APLICA
display: table-cell <td> <th> Se comporta como una celda de tabla td o th, su contenido y elementos secundarios se comportan como celdas de la tabla, de bloque. EN LA PRACTICA NO SE APLICA
display: table-row <tr> Se comporta como una fila de tabla tr, su contenido y elementos secundarios se comportan como celdas de la tabla, de bloque. EN LA PRACTICA NO SE APLICA
display: flex Se convierte en un contenedor flex. Por si solo se comporta como elemento de bloque. Elementos secundarios se convertiran en elementos de flexbox si los tuviera.
display: inline-flex Comparte propiedades tanto de linea como de elemento flex. Comportandose como texto simple que se inserta en un bloque de texto y sus elementos secundarios se convertiran en elementos flexbox.
display: grid Se convierte en un contenedor grid (rejilla). Por si solo se comporta como elemento de bloque. Elementos secundarios se convertiran en elementos de cuadricula (grid items) si los tuviera.
display: inline-grid Comparte propiedades de linea y de grid. Comportandose como texto simple que se inserta en un bloque de texto y sus elementos secundarios se convertiran en elementos de flexbox.

Propiedad visibility

No confundir con display:none
Visibility quita la visibilidad del elemento manteniendo el espacio que ocupa sin visualizar la caja.
Dos posibles valores: vissible y hidden (opacidad total) 

visibility: visible;   y    visibility.hidden;

Propiedad overflow

Objetivo controlar que ocurre con el contenido que desborda su caja contenedora, exite overflow-x, overflow-y, overflow-wrap y overflow.
Define cómo se muestra el contenido desbordado tanto en el eje horizontal como en el vertical.
Los posibles valores que puede tomar overflow son: visible, hidden, scroll, auto.
overflow: visible; El contenido desbordado es visible , mientras que el elemento en sí permanece en la altura especificada.
overflow: hidden; El contenido desbordante esta oculto y no se puede acceder a el.
overflow: scroll; El contenido desbordante es accesible gracias a una barra de desplazamiento vertical.
overflow: auto; El navegador decide si mostrar una barra de desplazamiento vertical o no.

Tamaño de caja

En el modelo de caja se tienen dos opciones para definir el tamaño de Caja mediante la propiedad box-sizing el cual por omision su valor es contentbox.
Para el contenedor con la clase box-sizing al que se define un ancho de 500px.
Con el valor content-box, padding de 20px y border-width de 5px. El ancho total seria 500 + 20 + 20 + 5 + 5 = 550px, no tomando en consideracion el valor de margin.
Con el valor border-box, padding de 20px y border-width de 5px. El ancho total seria 500 y el ancho del content seria de 500 - 20 - 20 - 5 - 5 = 410px. Garantizando con esto que el tamaño total de la caja sea igual al witdh y al height

Colapso de margenes verticales

Cuando dos bloques tienen sucesión y a ambos se les aplica margenes top y bottom los margenes se colapsan y solo acepta uno de ellos, es conveniente asignar margenes o solo top o solo bottom. Igual cosa suceder para contenedores hijos con respecto al contenedor padre los top o los bottom se colapsan aplicando el de mayor valor de margin top o bottom según el caso

Existe una regla dentro del modelo de caja que dice que los margenes verticales de un elemento con otro se van a acoplar

Adición de margenes horizontales

Los margenes horizontales si se aplican tanto left como right. No aplica el colapso en margenes horizontales

Centrado de cajas

Para esto al elemento de bloque en la clase que lo define se aplica las propiedades margin-left y margin-right con el valor auto para cada una de ellas. Si solo se aplica margin-left: auto entonces la caja se deplaza totalmente a la derecha no provocando el centrado, efecto similar a float-right. Si solo se aplica margin-rigth: auto entonces la caja se deplaza totalmente a la izquierda no provocando el centrado, efecto similar a float-left.

Posicionamiento CSS

Practicando con posicionamiento CSS

La propiedad float y position permiten modificar la posicion natural de cualquier elemento del documento html.
El navegador coloca cada elemento teniendo en cuenta el orden en el que aparece en el documento y su tipo de visualizacion dependiendo de si es un elemento de linea o de bloque.
Con float y position podemos modificar este comportamiento.

  • float Convierte un elemento en flotante desplazándolo hasta la zona más a la izquierda o más a la derecha de la posición en la que originalmente se encontraba. De existir varios float:left o float: right se apilan uno a continuacion de otro. Los margenes no se colapsan cuando son float
  • clear Limpia la flotación (left, right, both)
Position, puede aplicarse tanto a elementos block como a inline
Tipos de posicionamiento con cinco posibles valores:
  • static (default), respeta el flujo natural del posionamiento html
  • relative
  • absolute
  • fixed
  • sticky
Para mover los elementos posicionados se activan las propiedades (cinco propiedades):
  • top (vertical - eje Y)
  • bottom (vertical - eje Y)
  • left (horizontal - eje X)
  • right (horizontal - eje X)
  • z-index (profundidad - eje Z)
Las propiedades top y left van a tener preferencia por sobre bottom y right
Estos cinco propiedades no funcionan con el valor de static.

Posicionamiento static

Es el valor por defecto, significa que el elemento respeta el flujo HTML, por lo que no se puede mover ni posicionar.

Posicionamiento relative

Hace que el elemento conserve sus dimenciones y posición original en el flujo del documento. Si se mueve toma como referencia su posicion original. Un elemento se puede mover pero su espacio natural queda como referencia vacio.
top hace que baje la caja tantos pixeles desde su posicion original
left deplaza la caja tantos pixeles desde su posicion original hacia la derecha
bottom deplaza la caja tantos pixeles hacia arriba desde su posicion original, es anulado por top en caso de existir top
right deplaza la caja tantos pixeles desde su posicion original hacia la izquierda, es anulado por left en caso de existir left
Propiedades top, left, bottom o right pueden tener números negativos provocando el evento inverso.
Puede contener elementos con position: absolute en cuyo caso el item contenido se puede desplazar respecto al padre.
Para elementos inline o convertidos a display:inline pierde sus dimensiones.

Posicionamiento absolute

Con este valor de posicionamiento el elemento pierde sus dimenciones y posición original en el flujo del documento HTML. Para elementos inline lo convierte en displayL block. Si se mueve puede tomar como referencia uno de dos casos para los elementos:

  1. El primer ancestro con posicionamiento relativo o cualquier posicionado excepto posicionamiento static - caso en que este dentro de una caja con posicionamiento relativo, absolute, fixed o sticky
  2. Si no se encuentra un ancestro ancestro, se mueve respecto del documento HTML (Body)

Posicionamiento Fijo fixed

Los elementos fixed pierden sus dimensiones y posición original en el flujo del documento HTML. Si se mueve toma como referencia el elemento HTML y queda fijo en la posición, cuando el SCROLL se mueva, el elemento no lo hará, quedando FIJO.
Cuando su top, bottom, left y right se ubican en 0 genera un efecto de estiramiento util para menus ocultos y ventanas modal. Para elementos inline los transforma en elemento de bloque si tuviera width y height.

Posicionamiento Pegajosa sticky

Es una combinación de posicionamiento relative y fixed, para este valor de posicionamiento las propiedades de top, left, bottom y right, no mueven el elemento, sirven como un punto de referencia, mientras no lleguen a ese valor, se comporta como relativo, cuando llega se convierte en fixed.
Para que este comportamiento funcione el elemento sticky debe ser hijo directo del body o su elemento contenedor debe tener dimensiones definidas y solo será sticky dentro de las dimensiones de su contenedor padre.

Propiedad z-index

Es un propiedad que permite controlar la profundidad de los elementos posicionados distintos de static, su valor por defecto es auto, acepta valores positivos, negativos y el cero.
A mayor valor el elemento estará más al frente, a menor valor más al fonto.
Un elemento padre nunca podrá estar sobre sus elementos hijos, sin embargo los elementos hijos si pueden dándoles un valor negativo y que el padre no tenga z-index aplicado.
Dos elementos con posicionamiento y con el mismo z-index o posicionados sin z-index, se pone encima el que se renderiza después.

Contexto de apilamiento

Un contexto de apilamiento se forma cuando:

  • html es un contexto de apilamiento
  • Posiciono un elemento con position relative, absolute, fixed o sticky y se aplica un z-index, entonces se crea un contexto de apilamiento.
  • Cuando se usa propiedad opacity con valor distinto de 1.

Colores en CSS

En el buscador de google si se busca "rgb to hex" aparece un convertidor de colores.
Modo nombre por la referencia de un nombre de color en la lista de nombres se encuentra agrupados de acuerdo al tono base que se puede consultar en el sitio Nombres de los colores, Español en el cual permite obtener cualquiera de los 140 nombres de colores que aceptan los navegadores en tres columnas se obtiene el nombre, codigo hex y codigo rgb.
Para el modo hexadecimal el color se asigna con el signo # y el trio de codigos para Red, Green y Blue, ej. #4F3BA4, que tendría un valor de 4F para rojo, 3B para verde y A4 para azul. si adicional se desea agregar opacidad entonces se tienen dos codigos adicionales como por ej.: #4F3BA455 con esto se aplica una opacidad del 55%, con opacidad 0 implica sin color, con opacidad 100 implica color completo, oscila entre [0,100).
En el modo RGB el codigo se asigna mediante las funciones rgb(val_red, val_green, val_blue) donde los val oscilan entre 0 y 255, para el caso de agregar opacidad entonces funcion rgba(val_red, val_green, val_blue, val_opa), donde val_opa oscila entre [0, 1], donde 0 implica sin color y 1 color completo (sin opacidad), ej. rgba(218,165,32,.42) que aplica el color Goldenrod = #DAA52042 con opacidad del 42%.
HSL (HueSaturationLightness) (Tono, Saturación, Luminosidad)
El hue se representa en grados: No requiere que se ubique el simbolo ° al asignar el valor

  • 0° => red
  • 60° => yellow
  • 120° => green
  • 180° => cyan
  • 240° => blue
  • 300° => magenta

En el modo HSL existen dos funciones hsl(val_hue, val_sat, val_lum) y la funcion hsla(val_hue, val_sat, val_lum, val_opa) como en el caso de rgba el val_opa va de [0, 1] y los porcentajes son decimales ej. .42 para el 42% de opacidad, siendo 0 transparente o ausencia de color y 1 color completo (sin opacidad).
Saturation - Intensidad del color, donde 100% es el color puro y 0% escala de grises. Se asigna el valor seguido del simbolo % incluso para definir el 0%.
Ligthness - Luminocidad del color, si se quiere que tienda hacia luz u oscuridad. Donde 0% -> oscuridad(negro), 50% -> color puro, 100% => luz(blanco). Se define con el simbolo % incluso para el 0%. Ej. hsl(0, 100%, 50%), refleja el color rojo

Valores transparent y currentColor -
transparent es usado para eliminar el color previamente asignado ej. background-color: transparent; o puede asignarse rgba(0, 0, 0, 0) que seria equivalente.
currentColor Hereda el valor por defecto o asignado de la propiedad color (casi siempre negro, color del texto) del ancestro que tenga definida esta propiedad, busca del ancestro más cercano al más alejado.
Propiedad opacity Afecta a todas las propiedades de color de la clase donde se define, background-color, color y border-color. También se puede aplicar a una imagen con el mismo efecto.

Variables y Funciones en CSS

Custom Properties (Variables CSS) y Función var()

Las variables tienen un alcance o ambito en el que se la define (Selector) y toda la cadena de elementos subsecuentes. Los nombres de variables deben iniciar con dos guiones ej. 

    --bg-primary-color: rgba(125,125,125,.62);
Para usar una variables se usa la funcion var(), ej. background-color: 
    var(--bg-primary-color)   var(--bg-primary-color, orange)
 La funcion var con dos parámetros, el primero es la variable y el segundo es el valor que se aplicaría en caso de no existir la variable previamente con valor asignado, esto en los casos que se encuentre establecida en otro ambito o no se haya declarado la variable previamente.
Para definir un ambito global se la define en la pseudoclase que tiene mayor especificidad que la etiqueta html. 
   :root { ... }
.

Función url()

Ayuda a especificar la ruta que se desea cargar, similar al src usando en selector img o href del selector de enlace a.

Función calc()

Utilizada para realizar calculos aritméticos. Se debe obligatoriamente dejar espacio entre operadores y operandos. Se pueden realizar operaciones mezclando unidades como en los casos resaltados 


          .ch-10 {
            --padding-size: 2ch;
            --num-ch: 10ch;
            background-color: var(--default-bg-color);
            padding: var(--padding-size);
            witdh:calc( var(--num-ch) + var(--padding-size) * 2 );
          }
          .el {
            margin: 10px calc(2vw + 5px);
            border-radius: 15px calc(15px / 3) 4px 2px;
            transition: transform calc(1s - 120ms);
            background: #1E88E5 linear-gradient(
                to bottom,
                #1E88E5,
                #1E88E5 calc(50% - 10px),
                #3949AB calc(50% + 10px),
                #3949AB
            );
            transform: rotate(calc(10deg * 5));
            transform: rotate(calc(1turn + 45deg));
            animation-delay: calc(1s + 15ms);
          }

Función min() & max()

Son funciones que de una lista de valores separados por comas toma el valor mínimo o el valor máximo según el nombre de la funcion invocada min o max, puede generar un efecto de cambio dinámico. 


          .min-max { 
            width: min( 300px, 20vw, 20rem); 
            height: max( 200px, 25vh); 
          }

Función clamp()

Es una función que va a recibir tres valores: 1 El tamaño mínimo, 2 El tamaño ideal, 3 El tamaño máximo.
Permite hacer cálculos dentro del paso de los parámetros sin necesidad de usar calc()
Donde se puede aplicar: En responsive design. 


          h1 {
            font-size: clamp( 2rem, 1rem + 3vw, 3rem);
          }

Border y Outlines

Propiedad border-radius

Bordes redondeados - Existe el shorthand border-radius y los shorthands border-top-left-radius, border-top-rigth-radius, border-bottom-rigth-radius y border-bottom-left-radius. El shorthand border-radius empezaría en border-top-left-radius y seguiría en el sentido reloj. Si se aplican dos valores entones sería las diagonales top-left y bottom-rigth para el primer valor y top-rigth junto con bottom-left para el segundo valor. En el caso de recibir tres valores, entonces, el primer valor es para la top-left, el segundo valor es para la diagonal (top-right, bottom-left) y el tercer valor es para bottom-right. Si el contenedor es un cuadrado y se redondea a los cuatro lados con el 50% se obtiene un circulo perfecto. 


        .border-radius {
          background-color: lightslategray;
          width: 50%;
          height: 40vh;
          margin-left: auto;
          margin-right: auto;
          padding: 1rem;
          border: thick solid red;
          /* border-radius: 2rem; */
          border-radius: 1rem 4rem 2rem;
          border-top-left-radius: unset;
        }        
          


      .border-radius-circulo {
        background-color: black;
        width: 5rem;
        height: 5rem;
        margin-left: auto;
        margin-right: auto;
        margin-top: 1rem;
        border-radius: 50%;
      }

Border elípticos - Se realiza mediante el establecer 2 valores por cada uno de los parámetros border-top-left-radius, border-top-rigth-radius, border-bottom-right-radius y border-bottom-left-radius, los juegos de valores separar por el caracter /. Antes del / iran todos los que correspondan a los radios y luegos del / todos los que corresponden a la formacion elíptica, siendo el primer valor para x y el segundo para y. Es decir que si en x y y son iguales entonces es un borde redondeado el que se aplico. 


        .border-eliptico {
          background-color: lightslategray;
          width: 50%;
          height: 40vh;
          margin-left: auto;
          margin-right: auto;
          margin-top: 1rem;
          padding: 1rem;
          border-radius: 4rem 4rem 2rem / 2rem 1rem 2rem;
        }

Propiedad outline

Se define como un borde externo, existe la propiedad shorthand border-outline. Existiendo outline-offset, outline-width, outline-style y outline-color. Similar a border. No forma parte del modelo de caja. El offset significa distancia de alejamiento de la caja, pudiendo ser positivo o negativo, despues o antes del borde. Viene aplicado por default en los input y pueden ser modificados. 


        .outlines {
          border: thick solid red;
          outline-offset: 1rem;
          outline-offset: -0.81rem;
          outline-width: 3px;
          outline-style: groove;
          outline-color: #0054dc;
        }

Estilos del Fondo

Propiedad background-color

Definir el color de fondo del elemento

Propiedad background-image

Definir la imagen de fondo del elemento. se implementa usando la funcion url()

Propiedad background-size

Establecer el tamaño de la imagen de fondo del elemento, primer valor es el x, segundo valor es el y.
* Cover .- Escala la imagen al mayor tamaño posible sin estirarla. Si las proporciones de la imagen difieren de las del elemento, es recortada vertical u horizontalmente para que no quede un espacio vacío. Cambia el tamaño de fondo para asegurarse de que permanezca completamente visible. Truncando el resto de su contenido. Cubre toda el elemento pero no asgura que se vea toda la imagen
* Contain .- Escala la imagen al mayor tamaño posible sin recortarla ni estirarla. Cambia el tamaño de la imagen de fondo para asegurarse de que el elemento esté completamente cubierto. Si se establece un solo valor entonces lo asume como el áncho y el alto lo toma automáticamente en relación al width, es decir, no se deforma y puede repetirse.

Propiedad background-repeat

Definir como se repite la imagen de fondo en el elemento. Por default el valor es repeat. Posibles valores repeat-x, repeat-y, no-repeat, space (deja un espacio entre una y otra repeticion), round. Es un shorthand de las propiedades background-position-x y background-position-y.

Propiedad background-position

Definir la posición de la imagen de fondo, primer valor eje x, segundo valor eje y. Si no se especifica el segundo valor este será center. Aparte de valores numéricos se puede indicar posicionamiento con las palabras: center, top, bottom, left y/o right. background-position: calc(78vw - 300px) top;

Propiedad background-clip

Definir cuanto debe externderse el fondo dentro del elemento. border-box (por default - inicia la imagen desde el fin del borde hacia el contenido), padding-box (inicia la imagen desde el inicio del borde o desde el inicio del padding hacia el contenido), content-box (la imagen inicia desde el contenido). Algunas veces puede verse un background-color “leak” salirse de un borde cuando border-radius esta presente (here’s a perfect example). Para prevenir esto puede usarser la propiedad CSS background-clip como en la clase .element: 

          
          background-clip: content-box;
          background-clip: border-box;
          background-clip: padding-box;

          .element {
            border-radius: 10px;

            /* Prevent background color leak outs */
            -webkit-background-clip: padding-box; 
            -moz-background-clip:    padding; 
            background-clip:         padding-box;
          }

Propiedad background-origin

Definir el origen de la imagen de fondo. Tiene los mismos valores que background-clip. padding-box (por default - inicia la imagen desde el inicio del borde o desde el inicio del padding hacia el contenido), border-box (inicia la imagen desde el fin del borde hacia el contenido), content-box (la imagen inicia desde el contenido) 


        background-origin: padding-box;
        background-origin: content-box;
        background-origin: border-box;

Diferencias entre clip y origin

Propiedad background-attachment

Definir como se comportará la imagen de fondo al desplazarse por la página. Valor por defecto es scroll. Con el valor fixed la imagen se posicione en el viewport y conforme nos movemos la imegen queda fija generardo efecto del espacio en la htmal como una ventana con vista a imagen de fondo.

Propiedad background

Es el shorthand de las ocho propiedades anteriores, en lo posible evitar usar la propiedad background como tal y usar cada una de las anteriores. Se define en el orden background-image background-position background-size background-repeat background-origin background-clip background-attachment y background-color.

Fondos Multiples

Con la propiedad background-image se pueden agregar tantas url para definir varias imagenes separadas por comas background-image: url("img01.jpg"), url("img02.jpg"), url("img03.jpg"); quedando visible la primera img01.jpg y de ultima queda más al fondo. Asi también la propiedades definibles en la propiedad shorthand background como background-repeat: repeat, no-repeat, repeat-x; en caso que se desee que un mismo valor afecte a varias imagenes solo se define un valor ej.: background-size: contain; , otro ej.: background-size: 10% contain; Enlace de práctica de composición de imagenes de fondo, opacidad y texto sobrepuesto imagen-text-opacity para Backgraund. 
          
          /* Basta representar un div vacio con la clase .bg-art para representar composición de imagenes de la previa*/
          .bg-art {
            width: 1000px;
            height: 700px;
            background: url("../assets/img/arbol.png") no-repeat center bottom,
              url("../assets/img/aves.png") no-repeat center bottom fixed,
              url("../assets/img/cielo.png") no-repeat center top;
            background-size: contain;
            background-attachment: fixed;
          }

Diseñando una card o tarjeta 👉 vista de imagenes con tamaños específicos

object-fit: para una img o un video el como se redimensiona para ajustarse al contenedor (Similar a background-size).
object-fit: fill | contain | cover | none | scale-down
fill (default) objeto se estira y cubre caja, no mantiene aspecto. contain objeto mantiene relacion de aspecto, ajustandose al tamaño de caja de acuerdo a alto o a ancho. cover objeto se recorta para ajustar a caja, mantiene aspecto. none objeto no se redimensiona, mantiene aspecto y recorta contenido de acuerdo a caja. scale-down Contenido se dimensiona como si none o contain estuvieran especificados tomando el menor de ellos posible, lo que resultaría en un tamaño de objeto concreto más pequeño. si se aplica posteriormente none o contain tiene prioridad, debiendose eliminar scale-down.

object-position: determina el alineamiento del objeto dentro de caja (similar a background-position) con coordenadas x, y puede ser un valor (px o %) o una palabra x(left center right) y(top center bottom. Por default es 50% 50% en x y y.


    object-position: <position>
    <position> = 
      [ left | center | right ] || [ top | center | bottom ]  |
      [ left | center | right | <length-percentage> ] [ top | center | bottom | <length-percentage> ]?  |
      [ [ left | right ] <length-percentage> ] && [ [ top | bottom ] <length-percentage> ]

Texto en columnas

Ubicar listas como columnas, multi-column-layout. Se basa en cuatro propiedades column-count cuantas columnas se usaran, column-gap espacio de separación mínimo entre una y otra columna, column-rule funciona como border o como outline (grosor tipo color) se nota cambios entre list-style-position inside y outside con esta propiedad, column-width Ancho minimo para cada columna, puede ser auto.
En caso de un article que contiene 6 parrafos se podría ubicar en tres columnas de manera similar. 

    .text-column-4 {
      column-count: 4;
      column-gap: 2rem;
      column-rule: thick solid #2a3132;
      column-width: 100px;
      list-style-type: lower-greek;
      list-style-position: inside;
    }
